# Mosaic Ideation

## Purpose

Generate controlled design directions for a human mosaic artist.

## Do not

- Produce only generic image prompts.
- Treat the mosaic as a pixel grid.
- Generate tiny details that cannot survive in broken tile.
- Ignore text readability.
- Pretend concepts are physically executable unless an execution compiler exists.

## Generate concepts as studio directions

Each concept should include:

- name;
- artistic intent;
- composition description;
- major regions;
- locked elements;
- flexible elements;
- mosaic grammar: shard size, flow, border logic, text logic;
- palette subset;
- image prompts;
- critique hooks.

## Concept diversity

Return materially different concepts. Prefer three from this list:

- landscape-led;
- typography-led;
- symbolic/ceremonial;
- abstract color-flow;
- community/children/path-led;
- ecological/local flora-led.

## Mosaic grammar reminders

- Text needs large, stable strokes and high contrast.
- Background can tolerate irregular fragments.
- Arcs and paths need shard-flow alignment.
- Focal symbols should have clean silhouettes.
- Dark outline tiles can save readability but may make the piece cartoonish if overused.
